# Claw Calendar Skill v4.1 更新总结

## 更新时间
2026-04-09 12:20 UTC

## 核心变更

### 新逻辑架构

**v4.0（旧）**:
```
Cron 触发 → LLM 调用 calendar_push_reminder → LLM 调用 message 工具 → 推送
```

**v4.1（新）**:
```
Cron 触发 → LLM 生成文案（不调用工具） → delivery 自动转发 → 推送
```

### 关键变化

| 项目 | v4.0 | v4.1 |
|------|------|------|
| **LLM 任务** | 调用 `calendar_push_reminder` + `message` | 只生成文案 |
| **推送方式** | LLM 调用工具 | delivery 自动转发 |
| **可靠性** | 依赖 LLM 工具调用 | delivery 系统保证 |
| **Payload** | 简单文本 | 完整指令（包含文案要求） |
| **Session** | isolated | isolated |
| **Delivery** | 无配置 | `mode: announce, channel: xx, to: xx` |

## 工作流程

### 创建事件
```
1. 用户：10 分钟后提醒我喝水
   ↓
2. LLM 调用 calendar_append_plan({
     title: "喝水",
     reminderStages: [{ offset: 10, offsetUnit: "minutes" }]
   })
   ↓
3. createReminderCron 为每个阶段 × 每个渠道创建 Cron
   - payload.message: 完整指令（生成文案，不调用工具）
   - delivery: { mode: 'announce', channel: 'xx', to: '...' }
   - session: 'isolated'
   - delete-after-run: true
   ↓
4. Cron 任务保存到 /root/.openclaw/cron/jobs.json
```

### Cron 触发
```
1. Cron 触发时间到达
   ↓
2. isolated session 启动
   ↓
3. LLM 收到 Payload 中的 message
   ↓
4. LLM 生成温馨文案（不调用任何工具！）
   ↓
5. LLM 回复文案
   ↓
6. delivery 自动捕获并发送到指定渠道
   ↓
7. 标记为 "delivered"
   ↓
8. Cron 任务自动删除
```

## 优势

1. **可靠性提升**: 不依赖 LLM 工具调用，delivery 系统保证推送
2. **简化 LLM 任务**: LLM 只需生成文案，无需关心推送细节
3. **代码清晰**: Cron 创建时即确定所有推送参数
4. **易于调试**: delivery 状态明确记录在 Cron 任务中
